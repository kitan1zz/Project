# ОБЗОР ПРОЕКТА

## 1. ПРОЕКТ OVERVIEW

Образовательный веб-сайт о блокчейне, NFT и экосистеме TON, созданный для людей без технического бэкграунда, особенно для школьников. Проект решает проблему сложности понимания криптовалютных технологий через визуализации, анимации и интерактивные демонстрации. Сайт объясняет базовые концепции блокчейна, работу NFT, процесс передачи владения (включая off-chain логику) и связь между TON и Telegram простым и понятным языком.

### Описание основных директорий:

- **app/** — содержит все страницы и маршруты приложения, используя Next.js App Router. Каждая поддиректория представляет отдельную страницу.
- **components/** — переиспользуемые React компоненты, включая навигацию, провайдер темы и визуализации.
- **app/api/** — серверные API маршруты для симуляции работы с NFT (без реального блокчейна).
- **public/** — статические файлы, доступные напрямую по URL.

## 4. BACKEND (МИНИМАЛЬНЫЙ)

### Описание логики:

Backend реализован через Next.js API Routes и использует in-memory storage (данные хранятся в памяти сервера, не в базе данных). Это упрощённая симуляция для демонстрации работы с NFT без реального блокчейна.

### API маршрут (`app/api/nft/route.ts`):

**GET `/api/nft`** — возвращает список всех NFT и транзакций
- Используется для получения текущего состояния

**POST `/api/nft`** — создаёт транзакцию передачи NFT
- Принимает: `{ nftId, from, to }`
- Валидирует, что отправитель является владельцем
- Создаёт транзакцию со статусом "pending"
- Симулирует обработку (задержка 1 секунда)
- Обновляет владельца NFT
- Меняет статус транзакции на "confirmed"
- Возвращает обновлённые данные

**DELETE `/api/nft`** — сбрасывает демо-данные
- Восстанавливает начальное состояние NFT и очищает транзакции

### Симуляция NFT владения/передачи:

1. **Хранение данных**: Массивы `nfts` и `transactions` в памяти сервера
2. **Создание транзакции**: При POST-запросе создаётся объект транзакции
3. **Валидация**: Проверка прав владения перед передачей
4. **Обработка**: Симуляция задержки сети (1 секунда)
5. **Обновление**: Изменение владельца NFT и статуса транзакции

**Важно**: Это демонстрация. В реальном блокчейне процесс включает:
- Подпись транзакции приватным ключом
- Отправку в сеть блокчейна
- Валидацию узлами сети
- Запись в блокчейн после подтверждения

## 5. ИНСТРУКЦИИ ПО ЗАПУСКУ И ДЕПЛОЮ

### Локальный запуск:

1. **Установите зависимости:**
```bash
npm install
```

2. **Запустите dev-сервер:**
```bash
npm run dev
```

3. **Откройте браузер:**
```
http://localhost:3000
```

4. **Сборка для продакшена:**
```bash
npm run build
npm start
```

### Деплой на Vercel:

#### Вариант 1: Через Vercel CLI

1. Установите Vercel CLI:
```bash
npm i -g vercel
```

2. Войдите в аккаунт:
```bash
vercel login
```

3. Деплой:
```bash
vercel
```

Следуйте инструкциям в терминале. Vercel автоматически определит Next.js проект.

#### Вариант 2: Через веб-интерфейс Vercel

1. Зайдите на [vercel.com](https://vercel.com) и войдите в аккаунт
2. Нажмите "Add New Project"
3. Импортируйте ваш Git-репозиторий (GitHub, GitLab, Bitbucket)
4. Vercel автоматически определит Next.js и настроит проект
5. Нажмите "Deploy"

После деплоя сайт будет доступен по адресу `https://your-project.vercel.app`

### Переменные окружения:

Для этого проекта не требуются переменные окружения, так как используется in-memory storage для демонстрации.

### Docker (опционально):

Docker не обязателен для этого проекта, так как Vercel предоставляет встроенную поддержку Next.js. Если нужен Docker для других целей, можно создать `Dockerfile`, но это не требуется для стандартного деплоя на Vercel.
